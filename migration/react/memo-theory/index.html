    Memo Theory

    When you're passing functions or objects as props in React, especially to memoized components, there are important performance and correctness considerations to keep in mind.

    üîÑ Why this matters:

    React uses reference equality to determine if props have changed. If you're passing a new object or function instance on every render, even if its contents are the same, it will break memoization and cause re-renders.

    üîç Considerations when passing functions/objects as props

    ‚úÖ 1. Memoize functions and objects before passing

    Use useCallback (for functions) or useMemo (for objects) in the parent component:

    const memoizedFunction = useCallback(() => {
  // do something
}, []);

const memoizedObject = useMemo(
  () => ({
    key: value,
  }),
  [value]
);

<BuySellAmountInfo myFn={memoizedFunction} config={memoizedObject} />;

    This prevents unnecessary re-renders due to prop changes.

    üö® 2. Avoid inline definitions

    <BuySellAmountInfo
  config={{ currency, rateCalculation }} // ‚ùå New object every render
  onChange={() => doSomething()} // ‚ùå New function every render
/>

    Even if currency and rateCalculation don't change, the object reference does ‚Äî so React will treat it as a new prop.

    üß† 3. Understand when memoization is worth it

    If your child component is:

    • Lightweight and doesn't render often ‚Üí you may not need React.memo.
    • Heavy or deeply nested ‚Üí memoization and stable props help performance.

    üì¶ 4. For form objects (like from useForm)

    These are typically already stable, so passing them directly is fine:

    const [form] = Form.useForm(); // ‚úÖ stable reference
<BuySellAmountInfo form={form} />;

    But don‚Äôt wrap in useMemo, it's unnecessary.

    ‚úÖ Best Practices Summary

    What You‚Äôre Passing Memoization Needed? How
    Plain values ‚ùå N/A
    Functions ‚úÖ useCallback(...)
    Objects (config, state) ‚úÖ useMemo(...)
    Ant Design form object ‚ùå (already stable) Pass directly

    üß™ Example

    const [form] = Form.useForm();

const config = useMemo(
  () => ({
    currency,
    rateCalculation,
  }),
  [currency, rateCalculation]
);

const handleChange = useCallback(() => {
  console.log("Changed");
}, []);

<BuySellAmountInfo form={form} config={config} onChange={handleChange} />;

    KNowing ThySelf

    Knowing whether React.memo and memoization are actually working is critical when you're optimizing React performance.

    Here‚Äôs how to check if React.memo and memoized props are effective:

    ‚úÖ 1. Use console.log() inside the component

    Add a log at the top of your component

    const BuySellAmountInfo = React.memo(function BuySellAmountInfo(props) {
  console.log('BuySellAmountInfo rendered');
  ...
});

    Then, in the parent component, update unrelated state or props.

    ‚úÖ If the log does not show, memoization is working.

    ‚úÖ 2. Use React Developer Tools (Profiler tab)

    Steps:
    1. Install React Developer Tools.
    2. Open the "Profiler" tab in DevTools.
    3. Click "Record", perform your UI interaction, and stop recording.

    4. Look for your component (BuySellAmountInfo):

      • If it's gray = didn't re-render ‚úÖ
      • If it's colored = it re-rendered üîÅ

    ‚úÖ 3. Use a custom areEqual function

    This helps if props are objects or arrays. Add a custom comparison and log inside it:

    function areEqual(prevProps, nextProps) {
  const isSame = prevProps.config === nextProps.config;
  if (!isSame) {
    console.log("Props changed, re-rendering");
  }
  return isSame;
}

export default React.memo(BuySellAmountInfo, areEqual);

    This shows when React decides to re-render based on props.

    ‚úÖ 4. Track re-renders visually (for dev/debug)

    Add a visual counter or color toggle:

    const renderCount = useRef(0);
renderCount.current++;

return <div>Rendered {renderCount.current} times</div>;

    5. ‚úÖ react-render-tracker (less popular)

    A lower-level utility for tracking component renders. Useful if you want to manually track and analyze renders but isn't as feature-rich as why-did-you-render.

    Tool Best For
    why-did-you-render ‚úÖ Detecting prop changes that trigger re-renders
    React DevTools Profiler üìä Visual performance insights
    console.log() + custom areEqual üß™ Manual checks (good for quick debugging)

    ‚ö†Ô∏è Common Pitfalls That Break Memoization

    Mistake Fix
    Passing inline objects/functions Use useMemo, useCallback
    Component relies on context or form instance Make sure it's stable or passed explicitly
    Expecting React.memo to prevent all re-renders It only skips re-renders when props don‚Äôt change by reference

    üîç Example: Effective Memoization

    const BuySellAmountInfo = React.memo(({ config }) => {
  console.log("Rendering BuySellAmountInfo");
  return <div>{config.currency}</div>;
});

const Parent = () => {
  const config = useMemo(() => ({ currency: "USD" }), []);

  return <BuySellAmountInfo config={config} />;
};

    The BuySellAmountInfo component will only re-render if config changes by reference, which it won't here due to useMemo.

    is this good or not but the issue in here in every form changes every component trigger

    What you have:

    You‚Äôre passing the same form instance plus some props to multiple child components inside a parent. The issue: every time the form changes, all components re-render.

    Why is that happening?
    • Each child component uses the same form instance.
    • When any form field changes, form updates internally.
    • If your child components call Form.useWatch([], form) or form.getFieldsValue() without fine-grained watching, all re-render.
    • Even if you memoize the components, the props change (or internal context changes) triggers re-render.
    How to fix or improve this

    1. Watch only specific form fields inside each component

      Use Form.useWatch to watch only the fields each component cares about, not the entire form.

      Example in `BuySellAmountInfo`
      const rateCalculation = Form.useWatch('rateCalculation', form);
const isMultiplicationStatus = Form.useWatch('isMultiplicationStatus', form);

      This way, the component only re-renders when those specific values change.

    2. Memoize your components

      Wrap them with React.memo() so they only re-render when props they actually depend on change.

    3. Avoid passing whole form or unrelated props

      If a child component doesn‚Äôt need the entire form, avoid passing the whole form.

      Instead, pass the specific values or handlers needed:

      <BuySellAmountInfo
    rateCalculation={rateCalculation}
    isMultiplicationStatus={isMultiplicationStatus}
    tolaranceValue={tolaranceValue}
/>

    4. Lift form state access to parent and pass only needed props

      Instead of having every child call Form.useWatch(form), have the parent watch needed fields and pass those as props. This centralizes state watching and limits re-renders.

    // In parent:
const rateCalculation = Form.useWatch("rateCalculation", form);
const isMultiplicationStatus = Form.useWatch("isMultiplicationStatus", form);

<BuySellAmountInfo
  rateCalculation={rateCalculation}
  isMultiplicationStatus={isMultiplicationStatus}
  tolaranceValue={tolaranceValue}
/>;

    TL;DR:

    • Watch only specific fields relevant to each component
    • Pass only those values as props
    • Memoize components to skip re-renders if props don‚Äôt change

    Memoize Theory SOLVED HOOKS

    import { useState } from "react";

const useFormSelectIds = () => {
  const [selectedIds, setSelectedIds] = useState({});

  const handleSelect = (value, field) => {
    setSelectedIds((prevState) => ({
      ...prevState,
      [field]: value,
    }));
  };

  const resetSelectedIds = () => setSelectedIds({});

  return { selectedIds, handleSelect, resetSelectedIds };
};

export default useFormSelectIds;

    should i use the useCallback here or in the component

    Yes ‚Äî you should use useCallback inside the useFormSelectIds hook to memoize the handleSelect function before returning it.

    ‚úÖ Why memoize inside the hook?

    If you don't memoize handleSelect, it will be a new function reference on every render, which will break React.memo on any component that receives it as a prop ‚Äî like your BuySellInfo.

    Memoizing it inside the hook keeps the function stable for all consumers of the hook ‚Äî and that‚Äôs the cleanest solution

    ‚úÖ How to fix it:

    Update your hook like this:

    import { useState, useCallback } from "react";

const useFormSelectIds = () => {
  const [selectedIds, setSelectedIds] = useState({});

  const handleSelect = useCallback((value, field) => {
    setSelectedIds((prevState) => ({
      ...prevState,
      [field]: value,
    }));
  }, []);

  const resetSelectedIds = useCallback(() => {
    setSelectedIds({});
  }, []);

  return { selectedIds, handleSelect, resetSelectedIds };
};

export default useFormSelectIds;

    ‚úÖ Benefits:

    • handleSelect and resetSelectedIds are now stable references
    • Components like BuySellInfo wrapped in React.memo will now properly skip re-renders unless other props actually change

    ‚ùå Don‚Äôt memoize it in the component that uses the hook

    Memoizing it outside the hook (in the component) doesn‚Äôt help because the hook already returns a fresh function each time unless it's memoized internally.
