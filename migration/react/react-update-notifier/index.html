    React App Update Notifier with meta.json and pnpm

    Great question ‚Äî aiming for the best and standard process is smart, especially for long-term maintainability and team workflows.

    Since you‚Äôre using:

    • React 18
    • pnpm
    • Ant Design
    • Likely a bundler like Vite, Webpack, or Next.js

    Let‚Äôs outline the best-practice approach to implementing a versioned update notifier ‚Äî including standards used in professional deployments.

    ‚úÖ Best Standard Process (2025 Edition)

    üß± Objective

    Notify users of a new version after deployment, using a meta.json file with:

    • App version (package.json)
    • Git commit hash (optional)
    • Build timestamp

    This approach:

    • Keeps builds traceable
    • Works with any CI/CD
    • Plays well with Vite, Webpack, Next.js
    • Doesn‚Äôt require a service worker (but can work with one)

    üõ†Ô∏è Step-by-Step Best Practice Setup

    1. Use a Build-Time Script (Prebuild Hook)

    Create a script that generates a meta.json file in public/.

    File: scripts/generate-meta.js

    const fs = require("fs");
const path = require("path");
const { execSync } = require("child_process");
const packageJson = require("../package.json");

const meta = {
  version: packageJson.version,
  buildTime: new Date().toISOString(),
  commitHash: execSync("git rev-parse --short HEAD").toString().trim(),
};

const outputPath = path.resolve(__dirname, "../public/meta.json");

fs.writeFileSync(outputPath, JSON.stringify(meta, null, 2));
console.log(`‚úÖ meta.json written to ${outputPath}`);

    //  package.json Setup (with pnpm + Vite)

    • ‚úÖ "type": "module" required for .mjs and ESM import
    • ‚úÖ prebuild runs before every pnpm build

    2. Add to pnpm Scripts

    In your package.json:

    {
  "scripts": {
    "prebuild": "node scripts/generate-meta.js",
    "build": "vite build" // or whatever build command you're using
  }
}

    ‚úÖ prebuild is a standard npm lifecycle hook, so this is idiomatic.

    3. Use useVersionChecker() Hook in React App

    (As shown earlier ‚Äî checks /meta.json and compares with the one at initial load.)

    You can place the hook in your App.tsx or root layout.

    ‚úÖ This is a client-side way to check if the backend or deployment has changed.

    4. CI/CD Tip (Optional but Recommended)

    In CI (like GitHub Actions, GitLab CI, etc.), ensure git is available and meta.json is committed or deployed.

    You can also inject version/commit as env vars and use them in Vite/Webpack if preferred.

    Why This Is Considered "Standard"

    ‚úÖ Feature Why It Matters
    Uses package version Automatically follows your semver
    Adds build timestamp Helps trace stale deployments
    Adds commit hash Helps developers debug precisely
    Runs on prebuild Clean, lifecycle-safe, cross-platform
    No service worker needed Lightweight and more predictable
    Works with any bundler Portable across Webpack, Vite, etc.

    üß† Alternatives

    Option When to Use
    Service Worker (onUpdate) If using PWA with offline support
    Vite plugin with define If you prefer to inject metadata at compile-time instead of via file
    Runtime /version API If you want the backend to tell the frontend what version is running

    üß© Recommendation for You

    Given you‚Äôre using:

    • React 18
    • Ant Design
    • pnpm
    • Likely Vite or Webpack

    ‚û°Ô∏è This meta.json + version hook approach is the best-practice standard.

    It‚Äôs framework-agnostic, lightweight, easy to test, and CI/CD friendly.

    ‚úÖ Use It in React App

    In your app (with Ant Design's notification), use the meta.json to detect version changes:

    // hooks/useVersionChecker.ts
import { useEffect, useRef } from "react";
import { notification } from "antd";

export function useVersionChecker(interval = 60000) {
  const currentVersion = useRef<string | null>(null);

  useEffect(() => {
    const check = async () => {
      try {
        const res = await fetch("/meta.json", { cache: "no-store" });
        const data = await res.json();
        if (!currentVersion.current) {
          currentVersion.current = data.version;
        } else if (data.version !== currentVersion.current) {
          notification.warning({
            message: "Update Available",
            description:
              "A new version of the app is available. Click to refresh.",
            duration: 0,
            onClick: () => window.location.reload(),
          });
        }
      } catch (e) {
        console.error("Meta version check failed:", e);
      }
    };

    check();
    const id = setInterval(check, interval);
    return () => clearInterval(id);
  }, [interval]);
}

    PACKAGE

    Yes, you can use a package to help detect and notify users about updates in a React web application ‚Äî especially when you're deploying SPAs where caching or service workers might cause users to stay on an outdated version.

    However, you don‚Äôt strictly need a package ‚Äî a simple polling of meta.json (as you've already implemented) is more efficient, customizable, and lightweight.

    That said, here are popular packages you can use if you prefer a ready-made solution:

    üß© Recommended Packages for Update Notifier

    TL;DR
    • If you want a solid, maintained, recommended package for update notifications in React apps, go with react-update-notification.
    • react-app-update-notifier is either deprecated, unmaintained, or just a different smaller package that isn‚Äôt recommended.

    1. üì¶ react-version-check

      What it does:

      • Polls a remote version file (like /meta.json)
      • Compares it to the current version
      • Lets you show a custom message/UI

      üîß Install:

      pnpm add react-version-check

      üß† Usage:

      import VersionCheck from "react-version-check";

if (!VersionCheck.isLatest()) {
  alert("A new version is available. Please refresh the page.");
}

      You can also add logic inside a useEffect or custom component.

      ‚ö†Ô∏è Downsides:

      • Not actively maintained
      • Assumes your version file is public and readable
      • Not as flexible as your current hook

    2. üì¶ react-app-update-notifier

      ‚úÖ What it does:

      • Polls a version endpoint
      • Provides a hook (useCheckUpdate) and state for UI control

      üîß Install:

      pnpm add react-app-update-notifier

      üß† Usage:

      import { useCheckUpdate } from "react-app-update-notifier";

const { hasUpdate, reloadPage } = useCheckUpdate({
  url: "/meta.json",
  interval: 60000,
  compare: (newMeta, oldMeta) => newMeta.version !== oldMeta.version,
});

if (hasUpdate) {
  return (
    <button onClick={reloadPage}>
      New version available! Click to update
    </button>
  );
}

      ‚úÖ Pros:

      • Nice encapsulated hook
      • Flexible config
      • Works like your own custom hook but is prebuilt

      ‚ö†Ô∏è Cons:

      • Adds an extra dependency for something you‚Äôve already solved
      • Might be overkill if you only need very basic logic

    3. üì¶ react-update-notification

      pnpm add react-update-notification
pnpm add -D react-update-notification

    Should You Use a Package?

    • | ‚úÖ Want simplicity and control | ‚û°Ô∏è Stick with your custom useVersionChecker.js |
    • | üß© Want ready-made abstraction | ‚û°Ô∏è Use react-app-update-notifier (best maintained option) |

    üîö TL;DR

    You can use packages like:

    Package Purpose Maintained? Notes
    react-version-check Checks version from a URL ‚ùå No Very minimal
    react-app-update-notifier React hook for update checking + UI ‚úÖ Yes Best option for hook-based UX

    But since you've already written a lightweight, clean, and fully customizable solution, you likely don‚Äôt need an extra package unless:

    • You want to support complex UI states out of the box
    • You want built-in comparison logic and polling with minimal code
