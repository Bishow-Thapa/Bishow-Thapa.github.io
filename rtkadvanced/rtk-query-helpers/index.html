    RTK

    RTK Query gives you several types of generated helpers, not just the hooks (useGetSomethingQuery) and the api.util helpers like prefetch.

    Let‚Äôs go through all of them systematically, so you know what each type is, when and why to use it.

    üß© RTK Query ‚Äútypes‚Äù or helper categories

    When you define an API slice like this:

    export const api = createApi({
  reducerPath: "api",
  baseQuery: fetchBaseQuery({ baseUrl: "/api" }),
  endpoints: (builder) => ({
    getPost: builder.query({
      query: (id) => `/posts/${id}`,
    }),
    createPost: builder.mutation({
      query: (data) => ({
        url: "/posts",
        method: "POST",
        body: data,
      }),
    }),
  }),
});

    RTK Query automatically generates different helper types:

    Type Example Description When to Use
    ü™ù Hooks useGetPostQuery(), useCreatePostMutation() Auto-generated React hooks for use in components 99% of the time in React components
    üß† Endpoint Definitions api.endpoints.getPost Access the definition object for programmatic dispatch Useful outside components (e.g., thunks, custom logic)
    ‚öôÔ∏è Utility Functions api.util.prefetch, api.util.invalidateTags, api.util.updateQueryData, api.util.resetApiState, etc. Programmatic helpers for caching, invalidation, and prefetching For advanced caching & data manipulation
    üß© Selectors api.endpoints.getPost.select(id) Select cached data directly from the Redux store For reading cache outside components (like in a thunk or a selector)
    üöÄ Initiate functions api.endpoints.getPost.initiate(id) Dispatch endpoint calls manually For fetching data imperatively or in non-React environments

    üîç Let‚Äôs dive into each one

    ü™ù 1. React Hooks

    Generated automatically for each endpoint.

    Query Example:

    const { data, isLoading, error } = useGetPostQuery(1);

    Mutation Example:

    const [createPost, { isLoading, isSuccess }] = useCreatePostMutation();

    ‚úÖ Use When:

    • In React components
    • You want automatic fetching, caching, revalidation, and state updates

    ‚öôÔ∏è 2. Utilities (api.util)

    The api.util object gives you lower-level control over RTK Query‚Äôs cache and behavior.

    Utility Description Example Use
    api.util.prefetch(endpointName, arg, options) Preload data in background Hover-to-prefetch, predictive loading
    api.util.invalidateTags(tags) Force re-fetch of queries that provide specific tags After a mutation updates data
    api.util.updateQueryData(endpointName, arg, updateFn) Manually modify cached data Optimistic updates
    api.util.resetApiState() Clear all RTK Query cache and metadata On logout or app reset
    api.util.selectInvalidatedBy(store, tags) See which queries are invalidated For debugging or custom refresh logic

    Example ‚Äî Prefetch:

    dispatch(api.util.prefetch("getPost", 1, { ifOlderThan: 30 }));

    Example ‚Äî Optimistic update:

    dispatch(
  api.util.updateQueryData("getPost", 1, (draft) => {
    draft.title = "Updated immediately";
  })
);

    ‚úÖ Use When:

    • You want fine control over cache behavior
    • You‚Äôre prefetching or updating cached data manually

    üß† 3. Endpoints Object (api.endpoints)

    This gives you access to the endpoint definitions themselves ‚Äî both for queries and mutations.

    Example:

    const getPostEndpoint = api.endpoints.getPost;
console.log(getPostEndpoint);

    This exposes:

    • getPost.initiate() ‚Äî for dispatching
    • getPost.select() ‚Äî for selecting cache
    • Metadata about the endpoint (like type, queryFn, etc.)

    ‚úÖ Use When:

    • You want to dispatch a query manually in non-React code
    • You‚Äôre in Redux logic or a thunk

    üöÄ 4. Initiate Functions

    Every endpoint has an .initiate() function, which dispatches a query or mutation manually. This is what the generated hooks use internally.

    Example:

    dispatch(api.endpoints.getPost.initiate(1));

    You can also await it if you unwrap the promise:

    const result = await dispatch(api.endpoints.getPost.initiate(1)).unwrap();
console.log(result);

    ‚úÖ Use When:

    • You need to fetch data outside React (e.g. inside a thunk or middleware)
    • You want manual control over timing or cancellation

    üß© 5. Selectors

    Each query endpoint automatically generates selectors to access cache in the Redux store directly.

    Example:

    const selectPost = api.endpoints.getPost.select(1);
const postData = useSelector(selectPost);

    postData will contain:

    {
  data: {...},
  status: 'fulfilled',
  isLoading: false,
  isError: false,
  ...
}

    ‚úÖ Use When:

    • You need to read cached data outside a component hook
    • You‚Äôre writing custom selectors or logic in Redux slice

    üß∞ Full List of Common RTK Query APIs

    Here‚Äôs the complete reference for what‚Äôs available:

    API Instance (from createApi)

    Property Purpose
    reducerPath Name of the slice in Redux
    reducer Add to your store
    middleware Add to your store
    endpoints Access endpoint definitions
    util Cache and data utilities

    Utilities

    Method Purpose
    api.util.prefetch() Preload query data
    api.util.invalidateTags() Invalidate cached queries
    api.util.updateQueryData() Modify cache manually
    api.util.resetApiState() Clear all cached data
    api.util.selectInvalidatedBy() Find invalidated queries

    Endpoint Object (api.endpoints.yourEndpoint)

    Method Purpose
    .initiate(arg) Dispatch a query/mutation manually
    .select(arg) Access cached result
    .matchFulfilled/Rejected/Started Middleware matchers for RTK listeners
    .useQuery, .useMutation The React hooks automatically generated

    üß† Practical Example Overview

    Task You‚Äôd Use Example
    Fetch data in component useGetPostQuery(id) React hook
    Fetch manually in thunk dispatch(api.endpoints.getPost.initiate(id)) Manual fetch
    Prefetch on hover api.util.prefetch() Background load
    Update cache optimistically api.util.updateQueryData() Modify cache
    Invalidate cache after mutation api.util.invalidateTags() Trigger re-fetch
    Read cached data manually api.endpoints.getPost.select(id) Selector

    ‚ö° TL;DR

    Category Example When to Use
    Hooks useGetUserQuery() In React components
    Utilities api.util.prefetch() Prefetch / update cache manually
    Endpoint objects api.endpoints.getUser Programmatic control
    Selectors api.endpoints.getUser.select(id) Read cache directly
    Initiate dispatches dispatch(api.endpoints.getUser.initiate(id)) Fetch outside React

    MISC

    üèÜ Most commonly used RTK Query helpers in React apps

    Helper Type Example How often used Why it‚Äôs important
    React hooks useGetPostQuery(id), useCreatePostMutation() Very frequently (used in almost every component that fetches data) Simplest way to fetch, cache, and subscribe to server data; handles loading, error, and caching automatically
    Selectors api.endpoints.getPost.select(id) Sometimes (mainly in advanced or performance-sensitive components) Lets you read cached data from the Redux store without triggering a hook re-render. Useful outside React hooks (e.g., Redux-only logic or selectors).
    api.util.prefetch dispatch(api.util.prefetch('getPost', 1)) Occasionally Preloads data before the user navigates ‚Äî improves UX in list/detail views (hover-to-prefetch, predictive loading).
    api.util.updateQueryData dispatch(api.util.updateQueryData('getPost', 1, draft => {...})) Moderate Optimistic UI updates; you can instantly reflect changes in the UI before the server responds.
    api.util.invalidateTags dispatch(api.util.invalidateTags(['Post'])) Moderate Forces cached queries to re-fetch after a mutation. Core for keeping data in sync without manually refetching.

    ‚ö° Rarely used / advanced helpers

    Helper Type Example Why rarely used
    Endpoint .initiate() dispatch(api.endpoints.getPost.initiate(id)) Usually hooks handle fetching, but useful in non-React code like thunks, middleware, or service workers.
    api.util.resetApiState() dispatch(api.util.resetApiState()) Only used on logout or full cache reset. Not common in day-to-day UI updates.
    .matchFulfilled/.matchRejected For listenerMiddleware Advanced use case for centralized event handling. Rare in typical apps.

    ‚úÖ TL;DR: ‚ÄúMust-know for React apps‚Äù

    1. Hooks ‚Üí useQuery and useMutation ‚Üí Core of almost all components.
    2. invalidateTags ‚Üí Keep cache in sync after mutations.
    3. updateQueryData ‚Üí Optimistic updates (common in interactive UIs).
    4. Prefetch ‚Üí Optional but improves UX in detail views.
    5. Selectors ‚Üí Optional; mostly in advanced Redux usage.

    Everything else (manual initiate, reset, matchers) is nice to know but rarely touched in day-to-day React app development.
