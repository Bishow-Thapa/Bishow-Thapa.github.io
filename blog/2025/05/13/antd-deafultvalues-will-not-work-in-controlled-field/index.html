    antd: deafultValues will not work in controlled field

    it occurs because in Ant Design Forms, you should not use defaultValue on inputs inside a controlled Form.Item. Instead, the correct way to set initial form field values is to use the initialValues prop on the parent <Form> component or the initialValue prop on the individual <Form.Item> (though the recommended approach is to use initialValues on <Form> for consistency).

    Warning: [antd: Form.Item] defaultValue will not work on controlled Field. You should use initialValues of Form instead. Component Stack:

    TabRollForm.jsx
    <Form.Item
  label="Fresh/Roll"
  name={DealFormEnum?.FRESH_ROLL}
  rules={[
    {
      required: isDevelopmentMode,
      message: "Please Select Fresh/Roll Type",
    },
  ]}
>
  <Select
    disabled
    allowClear
    showSearch
    filterOption={(input, option) =>
      option.children.toLowerCase().includes(input.toLowerCase())
    }
    placeholder="Select Fresh/Roll Type"
    // ISSUE HERE
    defaultValue={MoneyMarketDealTypeEnum?.ROLL}
  >
    <Option value={MoneyMarketDealTypeEnum?.FRESH}>Fresh</Option>
    <Option value={MoneyMarketDealTypeEnum?.ROLL}>Roll</Option>
  </Select>
</Form.Item>

    Explanation and Best Practice

    • Ant Design Forms are controlled components that internally manage form state.

    • Using defaultValue on input components inside Form.Item conflicts with this controlled behavior, causing the warning.

    • The correct way is to provide initial values via the initialValues prop on the <Form> component, which sets the default values for all fields at once.

    • Alternatively, you can use initialValue on individual <Form.Item>, but this is less recommended if you already use initialValues on

      .

    • This ensures the form state is properly initialized and managed by Ant Design's form system.

    How to Fix

    Instead of this (causes warning):

    <Form.Item name="username" label="Username" rules={[{ required: true }]}>
  <Input defaultValue="John" />
</Form.Item>

    Do this:

    <Form initialValues={{ username: "John" }}>
  <Form.Item name="username" label="Username" rules={[{ required: true }]}>
    <Input />
  </Form.Item>
</Form>

    Or if you want to set initial value on a single field (less common):

    <Form.Item
  name="username"
  label="Username"
  initialValue="John"
  rules={[{ required: true }]}
>
  <Input />
</Form.Item>

    Why?

    • defaultValue is for uncontrolled components - React sets the initial value but does not control it afterward.
    • Ant Design Form fields are controlled components - the form manages the value internally, so defaultValue is ignored and causes warnings.
    • Using initialValues or initialValue lets Ant Design initialize the form state correctly and keep it in sync.

    Additional Notes

    • If you load data asynchronously (e.g., from an API), you can update form values dynamically using form.setFieldsValue or by resetting the form with new initialValues.
    • Avoid mixing defaultValue with controlled form fields to prevent inconsistent UI and warnings.

    References from your search results

    • Ant Design official docs and examples recommend using initialValues on <Form> rather than defaultValue on inputs.
    • Stack Overflow and community articles emphasize that defaultValue on controlled fields causes warnings and does not behave as expected.
    • Using initialValues on <Form> or initialValue on <Form.Item> is the correct way to set initial data for controlled fields.

    Summary

    To resolve the warning:

    • Remove defaultValue from input components inside Form.Item.
    • Set initial form values using the initialValues prop on the <Form> component.
    • Optionally use initialValue on individual <Form.Item>, but prefer initialValues on <Form> for consistency.
    • Manage dynamic updates with form.setFieldsValue if needed.
