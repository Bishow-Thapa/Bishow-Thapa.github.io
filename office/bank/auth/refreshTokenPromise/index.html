    Error

    but the issue is in dashboard i have called two api it hits two api 401

    It sounds like you have a situation where two API calls from your dashboard both return a 401 Unauthorized status, and your token refresh logic is triggered twice concurrently. This can cause issues such as multiple refresh requests being sent simultaneously, race conditions, or inconsistent state updates.

    Problem Explanation

    When two API calls return 401 almost simultaneously, your current code triggers the token refresh logic twice independently. This leads to:

    • Multiple refresh token requests sent to the server
    • Possibly overwriting tokens incorrectly
    • Potential logout/reset state if one refresh fails while the other is still pending

    How to Fix: Implement a Token Refresh Queue or Lock

    You want to ensure that when a token refresh is already in progress, other API calls wait for the refresh to complete instead of triggering their own refresh requests.

    Approach 1: Use a Refresh Token Lock (Singleton Refresh)

    You can create a shared promise that represents the ongoing refresh request. When a 401 happens, check if a refresh is already in progress:

    • If yes, wait for that refresh promise to resolve and then retry the original request.
    • If no, start the refresh request and store the promise.
    Example Implementation
    let refreshTokenPromise = null;

async function handle401(api, apiStatus) {
  if (apiStatus !== 401) return;

  if (!refreshTokenPromise) {
    const refreshToken = store.getState().auth.refresh;
    const formData = queryString.stringify({
      grant_type: OAUTH_REFRESH?.GRANT_TYPE,
      client_id: OAUTH_REFRESH?.CLIENT_ID,
      client_secret: OAUTH_REFRESH?.CLIENT_SECRET,
      scope: OAUTH_REFRESH?.SCOPE,
      refresh_token: refreshToken,
    });

    refreshTokenPromise = axiosInstance.login
      .post(CONFIG.BASE_URL, formData, {
        headers: {
          "Content-Type": "application/x-www-form-urlencoded",
        },
      })
      .then((refreshResult) => {
        const { data } = refreshResult || {};
        if (data) {
          api.dispatch(updateRefreshToken(data));
          return data;
        } else {
          api.dispatch(logout());
          api.dispatch(resetState());
          throw new Error("Unable to refresh token. Logged out.");
        }
      })
      .catch((error) => {
        api.dispatch(logout());
        api.dispatch(resetState());
        throw error;
      })
      .finally(() => {
        refreshTokenPromise = null; // Reset lock after refresh completes
      });
  }

  // Wait for the refresh token request to complete
  const newTokenData = await refreshTokenPromise;
  return { data: newTokenData };
}
    How to Use
    • When you detect a 401, call handle401(api, apiStatus).
    • If a refresh is already in progress, the function waits for it to finish.
    • If not, it triggers a refresh and shares the promise.
    • After refresh, you can retry the original API requests with the new token.
    Additional Tips
    • Retry original requests after refresh: After refreshing the token, retry the failed API calls with the new token.
    • Centralize API calls: Use an axios interceptor or a centralized API handler to manage 401 errors and refresh logic.
    • Handle refresh token expiry: If refresh token is invalid or expired, force logout.
    • Debounce multiple refreshes: The lock pattern above effectively debounces multiple refresh calls.

    Refactored axiosBaseQuery with Refresh Token Lock

    js
    import queryString from "query-string";
import axiosInstance from "./axiosInstance";
import { store } from "@shared/store/store";
import { updateRefreshToken, logout } from "@features/auth/services/authSlice";
import { resetState } from "@features/notifications/services/chatSlice";
import logger from "@shared/utils/logger";
import { OAUTH_REFRESH, CONFIG } from "@shared/utils/config";

// Shared promise to track ongoing refresh token request
let refreshTokenPromise = null;

export const axiosBaseQuery =
  ({ baseUrl } = { baseUrl: "" }) =>
  async ({ url, method, meta, data, params, headers }, api) => {
    try {
      // Perform the API request
      const result = await axiosInstance.base({
        url: baseUrl + url,
        method,
        meta,
        data,
        params,
        headers,
      });

      if (!result || !result.data) {
        throw new Error("Invalid response structure from the server");
      }

      return { data: result.data, meta: { ...meta, method } };
    } catch (axiosError) {
      const apiStatus = axiosError?.response?.status;
      const detail =
        axiosError?.response?.data?.detail ?? "An unknown error occurred";
      const status = axiosError?.response?.data?.status ?? "Error";
      const title =
        axiosError?.response?.data?.title ?? "An unknown error occurred";
      const type = axiosError?.response?.data?.type ?? "Error";

      if (apiStatus === 401) {
        // If no refresh in progress, start one
        if (!refreshTokenPromise) {
          const refreshToken = store.getState().auth.refresh;
          const formData = queryString.stringify({
            grant_type: OAUTH_REFRESH?.GRANT_TYPE,
            client_id: OAUTH_REFRESH?.CLIENT_ID,
            client_secret: OAUTH_REFRESH?.CLIENT_SECRET,
            scope: OAUTH_REFRESH?.SCOPE,
            refresh_token: refreshToken,
          });

          refreshTokenPromise = axiosInstance.login
            .post(CONFIG.BASE_URL, formData, {
              headers: {
                "Content-Type": "application/x-www-form-urlencoded",
              },
            })
            .then((refreshResult) => {
              const { data } = refreshResult || {};
              if (data) {
                api.dispatch(updateRefreshToken(data));
                logger.info({ msg: "Token refreshed successfully", data });
                return data;
              } else {
                logger.error({ msg: "Unable to refresh token. Logging out." });
                api.dispatch(logout());
                api.dispatch(resetState());
                throw new Error("Refresh token invalid");
              }
            })
            .catch((error) => {
              logger.error({ msg: "Refresh token error: ", error });
              api.dispatch(logout());
              api.dispatch(resetState());
              throw error;
            })
            .finally(() => {
              refreshTokenPromise = null; // Reset lock when done
            });
        }

        try {
          // Wait for refresh to complete
          await refreshTokenPromise;

          // Retry the original request with new token
          // Update headers with new access token from store
          const newAccessToken = store.getState().auth.access;
          const retryHeaders = {
            ...headers,
            Authorization: `Bearer ${newAccessToken}`,
          };

          const retryResult = await axiosInstance.base({
            url: baseUrl + url,
            method,
            meta,
            data,
            params,
            headers: retryHeaders,
          });

          if (!retryResult || !retryResult.data) {
            throw new Error("Invalid response structure from the server");
          }

          return { data: retryResult.data, meta: { ...meta, method } };
        } catch (retryError) {
          // If retry fails, propagate error
          return {
            error: {
              status: retryError?.response?.status || 401,
              data: {
                type: "Unauthorized",
                title: "Failed to refresh token or retry request",
                detail: retryError.message,
              },
            },
          };
        }
      }

      // Handle 400 errors specifically
      if (apiStatus === 400) {
        return {
          error: {
            status: status,
            data: {
              type: type,
              title: detail,
              detail: title,
            },
          },
        };
      }

      // Default error return
      return {
        error: {
          status: status,
          data: {
            type: type,
            title: detail,
            detail: title,
          },
        },
      };
    }
  };
    Explanation
    • refreshTokenPromise: A module-level variable that holds the current refresh token request promise.
    • When a 401 occurs, if refreshTokenPromise is null, we start a refresh request and assign the promise.
    • If another 401 happens while refresh is in progress, it waits for the existing refreshTokenPromise.
    • After refresh completes, the original request is retried with the updated access token.
    • If refresh fails, the user is logged out and state reset.
    • This prevents multiple simultaneous refresh requests and ensures all failed requests retry properly.

    ???+ tip :Additional Recommendations"

    -   **Axios Interceptors**: Consider moving this logic into an axios response interceptor for cleaner separation.
-   **Token Expiry Handling**: Handle cases where refresh token itself is expired or invalid.
-   **Error Feedback**: Provide user feedback or UI notifications on logout or token expiry.
-   **Testing**: Test with concurrent API calls triggering 401 to confirm only one refresh happens.

    How do I calculate the expiration time for a JWT

    To calculate the expiration time for a JWT, you need to decode the token and examine the "exp" (expiration) claim, which is a Unix timestamp representing the exact time the token expires. The expiration time is typically given in seconds since the Unix epoch (January 1, 1970).

    Steps to Calculate JWT Expiration Time
    1. Decode the JWT: Use a JWT decoding tool or library (such as jwt.io or jwt-decode) to decode the token and extract the "exp" claim.
    2. Interpret the "exp" value: The "exp" claim is a timestamp in seconds since epoch that indicates when the token expires.

    3. Calculate remaining time: Subtract the current time (in seconds since epoch) from the "exp" value to get the remaining validity duration of the token.

      Remaining time (seconds) = exp ‚àí current Unix time

    4. Use the remaining time: This remaining time can be used to determine if the token is still valid or to set cache expiry or token refresh logic.

    Example

    If your token has:

    • "exp": 1746769329 (expiration timestamp)
    • Current Unix time is, say, 1746765000 seconds

    Then:

    • 1746769329 ‚àí 1746765000 = 4329 seconds remaining

    • This means the token will expire in approximately 1 hour and 12 minutes from now.

    Additional Notes

    • The "iat" (issued at) claim shows when the token was issued, but for expiration you focus on "exp".

    • Because clock differences between client and server can cause issues, it's common to subtract a small buffer (e.g., a few seconds or minutes) from the remaining time to avoid using an expired token.

    • If you want to cache the token, use the remaining time as the cache TTL (time-to-live).

    • Libraries and frameworks often provide utilities to decode and check expiration automatically.

    This method is standard across JWT implementations and is recommended for managing token lifetimes effectively
