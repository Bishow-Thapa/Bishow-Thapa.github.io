    antd chart

    Summary of Key AntV Graph and Network Packages
    • G6: Network and graph visualizations.
    • X6: Diagrams and flowchart visualizations.
    • L7: Geospatial data visualizations.
    • F2: Mobile-friendly charts and graphs.
    • G2: Grammar of graphics for general data visualization.
    npm install @ant-design/charts --save

# or

pnpm add @ant-design/charts

    height: 400,

  return <Line {...config} />;
};
export default Page;

    import { useMemo } from "react";

// Custom hook to filter currencies for buying and selling
const useFilteredCurrencies = (currencies, selectedCurrency, excludeValue) => {
    return useMemo(() => {
        return currencies
            .filter((currency) => currency.isActive && currency.value !== excludeValue)

                value,
                label,
            }));
    }, [currencies, excludeValue]);
};

    Tip
    • getLabelByValue function: This function takes a value (e.g., the id or value) and maps it to the corresponding label from the banks array. It is used in the final step to display the readable label rather than the id.
    • Displaying label in summary: In the final step (description5), instead of directly showing the value (e.g., formValues.payFrom), we use getLabelByValue(formValues.payFrom, banks) to get the label associated with that value.
    • Updating the form state: The form state (formValues) is updated by selecting values (e.g., payFrom, theyReceivedAt, etc.) using the onChange event handler inside each Select component.

    This approach allows you to keep the internal id values while presenting the user-friendly label in the UI.

    Advanced Code

    Yes, you can make the option?.name dynamic by passing the property name as an argument to the getLabelByValue function. This way, instead of hardcoding name, you can access any property dynamically by referencing the property name passed to the function.

    Here's how you can modify the code:

    const getLabelByValue = (id, options, property = "name") => {
    const option = options.find((option) => option?.id === id);
    return option ? option[property] : null;
};

export default getLabelByValue;
    1. property = 'name': The property parameter is added with a default value of 'name', which means if no property is passed, it will default to 'name'.
    2. option[property]: This accesses the dynamic property of the found option.
    Example Usage

    If you want to get the name:

    const label = getLabelByValue(1, options);  // Defaults to 'name'

    If you want to get a different property, such as description:

    const label = getLabelByValue(1, options, 'description');

    This makes the function flexible to return any dynamic property of the object within the options array.
