    Upload Images Type

    Question 
    <Upload
    listType="picture"
    accept="image/*"
    maxCount={1}
    beforeUpload={() => false}
    fileList={fileList}
    onChange={onHandleChange}
>
    • accept png and jpg but it acceps gif also

    Scenario

    You're encountering an issue where your <Upload> component accepts GIFs, even though you've specified:

    accept = "image/*";

    This syntax (image/*) tells the file input to accept any image format, which includes:

    • .png
    • .jpg / .jpeg
    • .gif
    • .webp, etc.

    To restrict it to only .png and .jpg, you need to explicitly list those MIME types or extensions.

    ✅ Fix

    Update your accept prop to:

    accept = ".png,.jpg,.jpeg";

    <Upload
  listType="picture"
  accept=".png,.jpg,.jpeg"
  maxCount={1}
  beforeUpload={() => false}
  fileList={fileList}
  onChange={onHandleChange}
>
  <Button icon={<UploadOutlined />}>Upload</Button>
</Upload>

    This way, GIFs and other formats like .webpwon't even be selectable in the file dialog.

    Let me know if you also want to validate on the beforeUpload hook (e.g. in case someone bypasses the dialog filter).

    CONSTANT

    BASIC

    Absolutely — defining a constant for accepted file types is a clean and reusable approach. You can store the accepted image types in a constant and reuse it wherever needed in your project.

    Example: Define Reusable Constant

    1. Create a constants file

      You can create a constants.js or uploadConstants.js file:

      // uploadConstants.js

export const ACCEPTED_IMAGE_TYPES = ".png,.jpg,.jpeg";

    2. Use it in your Upload component

      import { ACCEPTED_IMAGE_TYPES } from "./uploadConstants";

<Upload
  listType="picture"
  accept={ACCEPTED_IMAGE_TYPES}
  maxCount={1}
  beforeUpload={() => false}
  fileList={fileList}
  onChange={onHandleChange}
>
  <Button icon={<UploadOutlined />}>Upload</Button>
</Upload>;

    ✅ Bonus: Add MIME type validation (optional)

    You can also add validation in beforeUpload to guard against users bypassing the accept filter:

    const ACCEPTED_TYPES_ARRAY = ["image/jpeg", "image/png"];

const beforeUpload = (file) => {
  const isAcceptedType = ACCEPTED_TYPES_ARRAY.includes(file.type);
  if (!isAcceptedType) {
    message.error("Only JPG and PNG files are allowed!");
  }
  return isAcceptedType || Upload.LIST_IGNORE;
};

    Using beforeUpload

    Question

    if i donot use the beforeupload. using this

    const uploadType = 'IMAGES';
const { extensions, mimeTypes } = ACCEPTED_FILE_TYPES[uploadType];

    wont be problem.

    No, it won't be a problem at all — you're totally fine to use:

    const uploadType = "IMAGES";
const { extensions, mimeTypes } = ACCEPTED_FILE_TYPES[uploadType];

    even if you don’t use beforeUpload.

    Here's what happens:

    If you only use:

    <Upload
    accept={extensions} // e.g., '.png,.jpg,.jpeg'
    ...
>

    Then the native file dialog will filter files based on accept. That means:

    • The user won’t be able to select disallowed files via the dialog.
    • ✅ Works as expected for most use cases.

    ⚠️ Important caveat:

    accept is a UI-level filter only. It does not prevent drag-and-drop or programmatic file uploads.

    So while this is enough in most cases, users can still technically bypass it:

    • Drag in a .gif even if .gif isn't allowed.
    • Use browser dev tools to override restrictions.

    ✅ Recommended Practice:

    If you're confident that users won’t drag in incorrect files or you're in a trusted/internal environment, just using accept={extensions} is enough.

    But if it's a public-facing app or you want stronger control, it's best to also add:

    beforeUpload = { beforeUpload };

    with a check using mimeTypes.

    TL;DR

    Feature Native accept only With beforeUpload
    File dialog filtering ✅ Yes ✅ Yes
    Drag-and-drop protection ❌ No ✅ Yes
    Full control & custom validation ❌ No ✅ Yes

    Implementation resuable Validation

    Let's create a reusable helper function called getBeforeUploadValidator() that works with your ACCEPTED_FILE_TYPES constant and ensures files match the allowed MIME types.

    BASIC

    1. Your uploadTypes.js (or uploadConstants.js)

      // uploadTypes.js

export const ACCEPTED_FILE_TYPES = {
  IMAGES: {
    extensions: ".png,.jpg,.jpeg",
    mimeTypes: ["image/png", "image/jpeg"],
  },
  DOCUMENTS: {
    extensions: ".pdf,.doc,.docx",
    mimeTypes: [
      "application/pdf",
      "application/msword",
      "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
    ],
  },
  VIDEOS: {
    extensions: ".mp4,.mov",
    mimeTypes: ["video/mp4", "video/quicktime"],
  },
};

    2. Reusable Validator Helper

      You can create this in uploadUtils.js:

      // uploadUtils.js
import { message } from "antd";
import { ACCEPTED_FILE_TYPES } from "./uploadTypes";

export const getBeforeUploadValidator = (uploadType) => {
  const typeConfig = ACCEPTED_FILE_TYPES[uploadType];

  if (!typeConfig) {
    console.warn(`Unknown upload type: ${uploadType}`);
    return () => Upload.LIST_IGNORE;
  }

  const { mimeTypes, extensions } = typeConfig;

  return (file) => {
    const isValid = mimeTypes.includes(file.type);

    if (!isValid) {
      message.error(`Invalid file type. Only allowed: ${extensions}`);
    }

    return isValid || Upload.LIST_IGNORE;
  };
};

    3. Using it in your Component

      import { ACCEPTED_FILE_TYPES } from "./uploadTypes";
import { getBeforeUploadValidator } from "./uploadUtils";

const uploadType = "IMAGES"; // or 'DOCUMENTS', 'VIDEOS', etc
const { extensions } = ACCEPTED_FILE_TYPES[uploadType];

<Upload
  listType="picture"
  accept={extensions}
  beforeUpload={getBeforeUploadValidator(uploadType)}
  maxCount={1}
  fileList={fileList}
  onChange={onHandleChange}
>
  <Button icon={<UploadOutlined />}>Upload</Button>
</Upload>;
    Benefits of This Approach
    • 🔁 Reusable: One utility works across all upload types.
    • 💡 Clear: All valid types are defined in one place (ACCEPTED_FILE_TYPES).
    • 🛡️ Safe: Prevents invalid files via drag-and-drop or custom hacks.
    • 🧼 Clean: Easy to maintain or extend with new file types.

    Size Validation

    Update ACCEPTED_FILE_TYPES

    Add a maxSizeMB or maxSizeBytes field to each upload type:

    // uploadTypes.js

export const ACCEPTED_FILE_TYPES = {
  IMAGES: {
    extensions: ".png,.jpg,.jpeg",
    mimeTypes: ["image/png", "image/jpeg"],
    maxSizeMB: 5, // 5MB limit
  },
  DOCUMENTS: {
    extensions: ".pdf,.doc,.docx",
    mimeTypes: [
      "application/pdf",
      "application/msword",
      "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
    ],
    maxSizeMB: 10,
  },
  VIDEOS: {
    extensions: ".mp4,.mov",
    mimeTypes: ["video/mp4", "video/quicktime"],
    maxSizeMB: 50,
  },
};

    Update getBeforeUploadValidator()

    // uploadUtils.js
import { message, Upload } from "antd";
import { ACCEPTED_FILE_TYPES } from "./uploadTypes";

export const getBeforeUploadValidator = (uploadType) => {
  const typeConfig = ACCEPTED_FILE_TYPES[uploadType];

  if (!typeConfig) {
    console.warn(`Unknown upload type: ${uploadType}`);
    return () => Upload.LIST_IGNORE;
  }

  const { mimeTypes, extensions, maxSizeMB } = typeConfig;

  return (file) => {
    const isMimeTypeValid = mimeTypes.includes(file.type);
    if (!isMimeTypeValid) {
      message.error(`Invalid file type. Only allowed: ${extensions}`);
      return Upload.LIST_IGNORE;
    }

    const isSizeValid = file.size / 1024 / 1024 <= maxSizeMB; // Convert bytes to MB
    if (!isSizeValid) {
      message.error(`File must be smaller than ${maxSizeMB}MB`);
      return Upload.LIST_IGNORE;
    }

    return true;
  };
};

    Summary

    Feature Implementation
    File Type Check Via mimeTypes.includes(file.type)
    File Size Check Via file.size / 1024 / 1024 <= maxSizeMB
    Reusability Fully reusable with centralized config

    Option 1: Pass Dynamic maxSizeMB as a Param to the Validator

    This is the cleanest way if the size limit depends on external conditions (like props, user selection, etc.).

    🛠 Updated getBeforeUploadValidator (with optional override)

    // uploadUtils.js

export const getBeforeUploadValidator = (uploadType, options = {}) => {
  const typeConfig = ACCEPTED_FILE_TYPES[uploadType];

  if (!typeConfig) {
    console.warn(`Unknown upload type: ${uploadType}`);
    return () => Upload.LIST_IGNORE;
  }

  const { mimeTypes, extensions, maxSizeMB: defaultMaxSizeMB } = typeConfig;
  const maxSizeMB = options.maxSizeMB ?? defaultMaxSizeMB;

  return (file) => {
    const isMimeTypeValid = mimeTypes.includes(file.type);
    if (!isMimeTypeValid) {
      message.error(`Invalid file type. Only allowed: ${extensions}`);
      return Upload.LIST_IGNORE;
    }

    const isSizeValid = file.size / 1024 / 1024 <= maxSizeMB;
    if (!isSizeValid) {
      message.error(`File must be smaller than ${maxSizeMB}MB`);
      return Upload.LIST_IGNORE;
    }

    return true;
  };
};
