    Theory

    In Ant Design (antd), the <Steps> component starts indexing from 0 ‚Äî just like arrays in JavaScript.

    <Steps current={1}>
  <Step title="Step 1" />
  <Step title="Step 2" />
  <Step title="Step 3" />
</Steps>

    In this example:

    • current={1} means the second step is active (not the first).
    • Steps are indexed: Step 1 => index 0, Step 2 => index 1, etc.

    ‚úÖ Recommended Fix:

    Status UI Step Index (<Steps current={}>)
    VERIFY_REQUEST 0
    APPROVAL_REQUEST 1
    MID_OFFICE_REQUEST 2
    ONHOLD 3
    BACKOFFICE_VERIFICATION 4
    COMPLETE 5

    If you want the first visible step to be the active one when it's VERIFY_REQUEST, then adjust your mapping like this:

    So you should either:

    • Subtract 1 from setStep(n), or
    • Refactor the switch logic to use 0-based values to match Ant Design.
