    SASS Example Folder Structure:

    src/
│
├── assets/             # Images, fonts, icons, etc.
│
├── features/           # Feature-specific code
│   ├── dashboard/      # Feature 1
│   │   ├── components/
│   │   ├── dashboard.scss
│   ├── user/           # Feature 2
│   │   ├── components/
│   │   ├── user.scss
│
├── shared/             # Shared code
│   ├── components/     # Reusable components (buttons, modals, etc.)
│   ├── styles/         # Global styles (variables, mixins, etc.)
│   │   ├── _variables.scss
│   │   ├── _mixins.scss
│   │   ├── _global.scss
│
├── App.jsx             # Your main React component
├── index.js            # Entry point

    Details on Folder Structure

    1. Shared Folder (Global Styles)

    In the shared folder, you can maintain global styles, such as Sass variables, mixins, resets, and global styles that will be used across the application.

    src/shared/styles/
  ├── _variables.scss   # Global variables like colors, fonts, etc.
  ├── _mixins.scss      # Global mixins
  ├── _global.scss      # Global styles (reset, body, fonts, etc.)
  ├── _animations.scss  # Global animations

    2. Feature-Specific Styles

    Each feature can have its own scss file to store styles relevant to that feature. This keeps styles modular and scoped to the feature.

    src/features/dashboard/dashboard.scss   # Styles specific to the dashboard feature
src/features/user/user.scss             # Styles specific to the user feature

    Within each feature folder, you can also organize the components and any nested styles specific to that feature. For example:

    src/features/dashboard/components/      # Individual component styles for the dashboard
src/features/user/components/           # Individual component styles for the user

    3. Importing and Using Sass

    • Shared Styles: Import shared styles globally, usually at the root of the app or in the entry file (like App.jsx).
    src/App.jsx
    import "./shared/styles/global.scss"; // Global styles (reset, fonts, etc.)
import "./shared/styles/_variables.scss"; // Global variables (if needed)
import "./shared/styles/_mixins.scss"; // Global mixins
    • Feature-Specific Styles: Import the feature-specific styles in the corresponding component.
    src/features/dashboard/Dashboard.jsx
    import "./dashboard.scss"; // Importing feature-specific styles

    Best Practices for Managing Sass in a Feature-Based Structure

    1. Modularity: Keep feature-specific styles scoped to the relevant feature folder. This makes your app more modular and easier to manage as it scales.

    2. Shared Styles: Define global styles such as colors, font sizes, and mixins in the shared/styles/ folder. This avoids duplication and ensures consistency across features.

    3. Naming Conventions:

      • Use descriptive and consistent naming for styles. For example, if a feature is called dashboard, its styles could be named dashboard.scss.
      • Use underscores (_) for partial files in Sass (those that don't compile into standalone CSS files but are imported into other files), such as _variables.scss, _mixins.scss, and _global.scss.

    4. Avoid Global Overrides: While it's fine to have global styles for body, fonts, and layout, try to avoid defining styles that could unintentionally override styles within specific features. Instead, scope styles to features as much as possible.

    5. Nesting: If necessary, use Sass's nesting feature to scope styles to a specific class or element.

    /* src/features/dashboard/dashboard.scss */
.dashboard {
  background-color: $primary-color;
  .card {
    padding: 10px;
  }
}

    src/
├── assets/
├── features/
│   ├── dashboard/

│   │   │   ├── Card.jsx
│   │   │   ├── Card.scss
│   │   ├── dashboard.scss
│   ├── user/
│   │   ├── components/
│   │   │   ├── UserCard.jsx
│   │   │   ├── UserCard.scss
│   │   ├── user.scss
├── shared/
│   ├── components/
│   ├── styles/
│   │   ├── _variables.scss
│   │   ├── _mixins.scss
│   │   ├── _global.scss
├── App.jsx
├── index.js

    Conclusion

    In summary, the best practice for organizing your Sass files in a feature-based folder structure is:

    • Place global styles (variables, mixins, resets) in the shared/styles/ folder.
    • Organize each feature's styles within the features/{featureName}/ folder, keeping them scoped to the feature.
    • Import global styles once in your main App.jsx or entry file and feature-specific styles within their corresponding components.

    This approach keeps your styles modular, scalable, and easy to maintain as your project grows.

                        required: true,
                        message: "Please select Selling Currency!",
                        },
                    ]}
                    >
                    <Select placeholder="Select Selling Currency">
                        {currencies.map((currency) => (
                        <Select.Option
                            key={currency.value}
                            value={currency.value}
                        >
                            {currency.label}
                        </Select.Option>
                        ))}
                    </Select>
                    </Form.Item>

                    <Form.Item>
                    <Button type="primary" onClick={handleNext}>
                        Next
                    </Button>
                    </Form.Item>
                </Form>
                )
            }
            />

            {/* Step 2 */}
            <Steps.Step
            title="Step 2"
            description={
                currentStep === 1 && (
                <Form form={form} name="step2-form" layout="vertical">
                    <Form.Item
                    name="counterpartyName"
                    label="Counterparty Name"
                    rules={[
                        {
                        required: true,
                        message: "Please input Counterparty Name!",
                        },
                    ]}
                    >
                    <Input />
                    </Form.Item>

                    <Form.Item>
                    <Button type="primary" onClick={handleNext}>
                        Next
                    </Button>
                    <Button
                        type="default"
                        onClick={handlePrev}
                        style={{ marginLeft: 10 }}
                    >
                        Back
                    </Button>
                    </Form.Item>
                </Form>
                )
            }
            />

            {/* Step 3 */}
            <Steps.Step
            title="Step 3"
            description={
                currentStep === 2 && (
                <Form
                    form={form}
                    name="step3-form"
                    layout="vertical"
                    onFinish={onFinishSubmit}
                >
                    <Form.Item
                    name="address"
                    label="Address"
                    rules={[
                        { required: true, message: "Please input your address!" },
                    ]}
                    >
                    <Input />
                    </Form.Item>

                    <Form.Item>
                    <Button type="primary" htmlType="submit">
                        Submit
                    </Button>
                    <Button
                        type="default"
                        onClick={handlePrev}
                        style={{ marginLeft: 10 }}
                    >
                        Back
                    </Button>
                    </Form.Item>
                </Form>
                )
            }
            />
        </Steps>
        </>
    );
};

export default VerticalStepsForm;
