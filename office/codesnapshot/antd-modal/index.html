    antd modal

    Creating a reusable modal component that can dynamically accept forms and their functionalities is a great approach to enhance code maintainability and reusability. Here’s a standard practice to achieve this using Ant Design, React, and functional components.

    Create a Reusable Modal Component

    You can create a ReusableModal component that accepts props for the title, visibility, form components, and submission handling.

    // ReusableModal.js
import React from 'react';
import { Modal } from 'antd';

const ReusableModal = ({ visible, title, onCancel, onOk, children }) => {
    return (
        <Modal
            title={title}
            visible={visible}
            onCancel={onCancel}
            footer={null} // Control footer visibility from the parent
        >
            {children}
            <div style={{ display: 'flex', justifyContent: 'flex-end' }}>
                <button onClick={onCancel} style={{ marginRight: '8px' }}>
                    Cancel
                </button>
                <button onClick={onOk} type="primary">
                    Submit
                </button>
            </div>
        </Modal>
    );
};

export default ReusableModal;

    Use the Reusable Modal in Your Component

    You can then use this ReusableModal in your DashboardTodo component, injecting the form dynamically.

    import React, { useEffect, useState } from "react";
import { useDispatch, useSelector } from "react-redux";
import axios from "axios";
import { Form, Input, Button } from "antd";
import { dashboardTodoService } from "../../redux/features/dashboard/services";
import ErrorMessage from "../../components/ImageGallery/ErrorMessage";
import LoadingSpinner from "../../components/ImageGallery/LoadingSpinner";
import ReusableModal from "../../components/ReusableModal";
import ResuableCard from "../../components/Card/ResuableCard";
import { DownloadOutlined, PlusCircleTwoTone, CopyTwoTone } from "@ant-design/icons";

const DashboardTodo = () => {
const dispatch = useDispatch();
const { todos, hasTodos, loading, error } = useSelector((state) => state.dashboardTodo);

const [isModalVisible, setIsModalVisible] = useState(false);
const [form] = Form.useForm();

useEffect(() => {
    const source = axios.CancelToken.source();
    dispatch(dashboardTodoService(source.token));

    return () => {
        source.cancel();
    };
}, [dispatch]);

if (loading) {
    return <LoadingSpinner />;
}

if (error) {
    return <ErrorMessage message={error} />;
}

if (!hasTodos) {
    return <div>No Todos Available</div>;
}

const handleAdd = () => {
    setIsModalVisible(true);
};

const handleOk = () => {
    form
    .validateFields()
    .then(values => {
        console.log("New Todo:", values);
        // Dispatch addTodo action here

        // Close the modal
        setIsModalVisible(false);
        form.resetFields();
    })
    .catch(info => {
        console.log('Validation Failed:', info);
    });
};

    return (
        <>
            <ResuableCard
                title="Todo List"
                content={todos.map(item => <p key={item.id}>{item.title}</p>)}
                extra={[
                    <DownloadOutlined key="download" />,
                    <PlusCircleTwoTone key="add" onClick={handleAdd} />,
                ]}
                icon={<CopyTwoTone />}
            />

            <ReusableModal
                visible={isModalVisible}
                title="Add Todo"
                onCancel={() => setIsModalVisible(false)}
                onOk={handleOk}
            >
                <Form form={form}>
                    <Form.Item
                        label="Todo Title"
                        name="title"
                        rules={[{ required: true, message: 'Please input the todo title!' }]}
                    >
                        <Input />
                    </Form.Item>
                </Form>
            </ReusableModal>
        </>
    );
};

export default DashboardTodo;

    1. ReusableModal Component:

      • The ReusableModal accepts visible, title, onCancel, onOk, and children props.
      • This allows the parent component to control the modal's visibility and actions while allowing dynamic content via children.

    2. Using the Modal:

      • In the DashboardTodo component, the modal is displayed when the "Add" button is clicked.
      • The form inside the modal is defined using Ant Design’s Form component.

    3. Form Submission:

      • The handleOk function validates the form fields. Upon successful validation, it logs the new todo and resets the form fields.

    Standard Practices

    • Separation of Concerns: Keeping the modal and the form separate promotes reusability. The modal can be reused across different components with different forms.
    • Dynamic Content: By passing children to the modal, you can inject any form or content as needed.
    • Controlled Components: Using Ant Design's Form ensures validation and state management are handled effectively.
    • Clear Functionality: Define clear functions for handling modal visibility and form submission, making it easy to understand and maintain.

    Further Enhancements

    • Custom Footer: Allow passing a custom footer if you need different buttons or actions.
    • Error Handling: Implement better error handling and user feedback.
    • Styling: Enhance the modal’s styling as needed to fit your application’s design.

    react-router-dom

    To navigate from the /dashboard route to the /todo route when clicking an icon, you can use the useHistory hook from react-router-dom (for versions 5 and below) or useNavigate from react-router-dom (for version 6 and above). Here’s how to implement this in your application.

    Step 1: Install react-router-dom

    If you haven't already, ensure that react-router-dom is installed in your project:

    npm install react-router-dom

    Step 2: Set Up Routing in Your Application

    Make sure your main application file has routing set up. Here's a basic example using React Router:

    // App.js or your main component
import React from 'react';
import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';
import Dashboard from './Dashboard';
import Todo from './Todo'; // Your Todo component

const App = () => {
    return (
        <Router>
            <Switch>
                <Route path="/dashboard" component={Dashboard} />
                <Route path="/todo" component={Todo} />
            </Switch>
        </Router>
    );
};

export default App;

    Step 3: Modify the Icon Click Handler

    Now, in your DashboardTodo component, modify the icon's click handler to navigate to the /todo route.

    import { useNavigate } from "react-router-dom";

// Inside your DashboardTodo component
const DashboardTodo = () => {
    const navigate = useNavigate();

    const handleNavigateToTodo = () => {
        navigate("/todo"); // Navigate to the /todo route
    };

    return (
        <>
        <ResuableCard
            title="Todo List"
            content={todos.map(item => <p key={item.id}>{item.title}</p>)}
            extra={[
                <DownloadOutlined key="download" />,
                <PlusCircleTwoTone key="add" onClick={handleAdd} />,
                <CopyTwoTone key="navigate" onClick={handleNavigateToTodo} />, // Navigate on click
            ]}
            icon={<CopyTwoTone />}
        />
        {/* ... other components */}
        </>
    );
};

    If you're using version 5, you would do something like this:

    import { useHistory } from "react-router-dom";

// Inside your DashboardTodo component
const DashboardTodo = () => {
    const history = useHistory();

    const handleNavigateToTodo = () => {
        history.push("/todo"); // Navigate to the /todo route
    };

    return (
        <>
        <ResuableCard
            title="Todo List"
            content={todos.map(item => <p key={item.id}>{item.title}</p>)}
            extra={[
                <DownloadOutlined key="download" />,
                <PlusCircleTwoTone key="add" onClick={handleAdd} />,
                <CopyTwoTone key="navigate" onClick={handleNavigateToTodo} />, // Navigate on click
            ]}
            icon={<CopyTwoTone />}
        />
        {/* ... other components */}
        </>
    );
};

    1. Navigation Hook:

      • Use useNavigate (v6) or useHistory (v5) to programmatically navigate to different routes.

    2. Click Handler:

      • The handleNavigateToTodo function is called when the icon is clicked, redirecting the user to the /todo route.

    3. Icon Integration:

      • Attach the click handler to the desired icon in the extra prop of your ReusableCard.

    This setup will allow you to navigate seamlessly between the dashboard and todo components when the respective icons are clicked.

    so when to use and history.push.

    Using <Link> and history.push (or navigate in React Router v6) serves similar purposes—navigating between routes in a React application—but they are suited for different scenarios. Here’s a breakdown of when to use each:

    1. Static Navigation:

      • Use <Link> when you have static navigation elements, such as links in a navigation bar, menu, or list items.
      • It provides better accessibility, as it renders an anchor (<a>) tag.
      import { Link } from 'react-router-dom';

const Navigation = () => (
    <nav>
        <Link to="/dashboard">Dashboard</Link>
        <Link to="/todo">Todo</Link>
    </nav>
);

    2. Declarative Navigation:

      • <Link> is declarative and easier to read. It handles navigation without needing to manage state or event handling manually. It also automatically handles active styling when using NavLink.

    3. Multiple Links:

      • If you need multiple links on a page, <Link> is simpler and cleaner than managing click events for each.

    1. Programmatic Navigation:

      • Use history.push (v5) or navigate (v6) when you need to navigate based on certain conditions or events (e.g., after a form submission, button click, or an API response).
      const handleSubmit = () => {
    // Do some processing
    history.push("/success");
};

    2. Conditional Navigation:

      • When navigation depends on user actions, like authentication or responses from API calls, using history.push or navigate allows for more control.

    3. Dynamic Routes:

      • If you need to navigate to a route that includes dynamic parameters (e.g., user IDs), you can easily construct the path using string interpolation.
      const handleNavigateToUser = (userId) => {
    navigate(`/users/${userId}`);
};

    Summary

    • Use for simple, declarative navigation where no additional logic is required. It's suitable for static links in your UI.
    • Use history.push or navigate for programmatic navigation, especially when user interactions or conditions determine when to navigate. This allows for greater flexibility and control over the navigation process.

    Both methods can coexist in your application depending on the context, allowing you to choose the best approach for each scenario.
