    onMessage

    The onMessage callback in react-idle-timer is a crucial feature when you're using cross-tab synchronization (enabled by crossTab: true). This callback allows you to define how messages are sent and received across different browser tabs within the same session.

    Overview

    The onMessage function in your React code is part of the react-idle-timer library, which allows you to handle cross-tab or cross-window communication when one tab or window is idle or active.

    What is onMessage?

    The onMessage function is called when a message is sent to the other tabs that are sharing the same session (through localStorage or BroadcastChannel API in modern browsers). It is part of the cross-tab messaging system that allows you to broadcast custom events between tabs.

    The typical use case for onMessage is when you want to communicate something custom across tabs. This is useful for things like:

    • Updating UI state across tabs (for example, showing a notification in all tabs when one tab performs an action).
    • Communicating specific events between tabs (like user actions, notifications, or data updates).
    • Syncing actions like showing modals (e.g., showing the idle modal in the leader tab, but informing other tabs as well).

    Breakdown of onMessage:

    In the context of your code, onMessage is used to handle incoming messages across different tabs that are part of the same browser session.

    1. Purpose of onMessage:

      The onMessage function is executed when a message is sent from one tab to another using the message() function provided by useIdleTimer. In your case, whenever the Send Message button is clicked, it calls the message('foo', true) function, which sends a message across tabs.

    2. How it works:

      • When you invoke message('foo', true) (which is triggered by the button click), a message is sent to all other open tabs/windows that are using the same session.

      • The message is handled by the onMessage function you defined.

      • In the onMessage function, you are simply incrementing a counter (setCount(count + 1)), which keeps track of how many messages have been received from other tabs.

    In Detail:

    • message(): Sends a message across all tabs, which can be a string or any serializable data.
    • onMessage(): Listens for these messages and performs a side effect when a message is received. In this case, it increments the count state, which is displayed in the UI.
    Example Scenario:
    1. You have multiple tabs open, and each tab is running this React app.
    2. When you click the "Send Message" button in one of the tabs, the message('foo', true) function sends a message to all other open tabs.
    3. The onMessage function in each of the other tabs will be triggered, which increments the count state in each tab. This way, you can track how many messages each tab has received.
    Why use onMessage?
    • Cross-tab Communication: This is useful for handling state changes or synchronization across multiple tabs that are part of the same session. For example, if you want to update the state of an app across multiple tabs whenever an event occurs in one tab, you can use this mechanism.
    • Idle Timer Coordination: The react-idle-timer library uses this feature to synchronize idle times and active states across multiple tabs. If one tab becomes idle, other tabs can be notified and adjust their behavior accordingly.

    Use Cases for onMessage and Cross-Tab Communication

    1. Session Management Across Tabs:

      • Imagine a situation where you want to ensure that when a user logs out or changes their session status in one tab, all other tabs are immediately notified and can react accordingly (e.g., log out or refresh the state).

      • For example, if the user logs out in one tab, all other open tabs should log out immediately or update their session status.

    2. Idle State Syncing Across Tabs:

      • If you want to track the idle state of a user across multiple tabs (e.g., showing an "Idle" status on all tabs when the user is inactive on any tab), you can use onMessage to notify all other tabs when one tab goes idle.

      • Similarly, when a user becomes active in one tab, you can notify all other tabs, which would update their state to reflect that the user is active.

    3. Real-Time Notifications Across Tabs:

      • If you're building a real-time notification system (like chat apps, alerts, etc.), and you want notifications to appear across all open tabs, you can use this cross-tab messaging to trigger UI updates in all tabs when a message is sent or received.

    4. Synchronization of Timers Across Tabs:

      • When you have a countdown timer (e.g., for an idle timeout), and you want the countdown to be synchronized across all tabs, you can broadcast the remaining time to other tabs.

    Codepen

    import { useEffect, useState } from "react";
import { useIdleTimer } from "react-idle-timer";
import "./styles.css";

export default function App() {
  const [state, setState] = useState < string > "Active";
  const [remaining, setRemaining] = useState < number > 0;
  const [count, setCount] = useState < number > 0;

  const onIdle = () => {
    setState("Idle");
  };

  const onActive = () => {
    setState("Active");
  };

  const onMessage = () => {
    setCount(count + 1);
  };

  const { getRemainingTime, getTabId, isLeader, isLastActiveTab, message } =
    useIdleTimer({
      onIdle,
      onActive,
      onMessage,
      timeout: 10_000,
      crossTab: true,
      leaderElection: true,
      syncTimers: 200,
    });

  useEffect(() => {
    const interval = setInterval(() => {
      setRemaining(Math.ceil(getRemainingTime() / 1000));
    }, 500);

    return () => {
      clearInterval(interval);
    };
  });

  const tabId = getTabId() === null ? "loading" : getTabId().toString();
  const lastActiveTab =
    isLastActiveTab() === null ? "loading" : isLastActiveTab().toString();
  const leader = isLeader() === null ? "loading" : isLeader().toString();

  return (
    <>
      <h1>React Idle Timer</h1>
      <h2>Cross Tab</h2>
      <br />
      <p>Current State: {state}</p>
      <p> Tab ID: {tabId}</p>
      <p>Last Active Tab: {lastActiveTab}</p>
      <p>Is Leader Tab: {leader}</p>
      <p>{remaining} seconds remaining</p>
      <p>{count} messages received</p>
      <button onClick={() => message("foo", true)}>Send Message</button>
    </>
  );
}

    Imagine a real-time chat application where multiple tabs (or users) are chatting simultaneously. We can use the onMessage feature to keep all tabs updated with new messages and typing status.

    Scenario:

    • When a user types a message in one tab, we want all other open tabs to see the message immediately and show the ‚ÄúUser is typing‚Äù status.
    Example
    import React, { useState, useEffect } from "react";
import { useIdleTimer } from "react-idle-timer";

const ChatApp = () => {
    const [messages, setMessages] = useState<string[]>([]); // Chat messages
    const [isTyping, setIsTyping] = useState(false); // User typing status
    const [currentMessage, setCurrentMessage] = useState(""); // Message being typed

    const onIdle = () => {
        setIsTyping(false); // Reset typing status when idle
    };

    const onActive = () => {
        // Optional: Reactivate something when the tab becomes active
    };

    const onMessage = (msg: string, fromLeader: boolean) => {
        // Handle messages from other tabs
        if (msg.startsWith("TYPING:")) {
            setIsTyping(true); // Mark the user as typing
        } else if (msg.startsWith("MESSAGE:")) {
            setMessages((prevMessages) => [...prevMessages, msg.substring(8)]); // Add new message to the chat
        }
    };

    const { message, reset, isLeader } = useIdleTimer({
        onIdle,
        onActive,
        onMessage, // Handle cross-tab messages
        timeout: 10000, // 10-second idle timeout
        crossTab: true, // Enable cross-tab messaging
        leaderElection: true, // Only one tab (leader) should send messages
    });

    const handleSendMessage = () => {
        if (isLeader()) {
            // Send the message to all other tabs
            message(`MESSAGE:${currentMessage}`, true);
        }
        setCurrentMessage(""); // Reset input field
    };

    const handleTyping = () => {
        if (isLeader()) {
            // Notify other tabs that the user is typing
            message("TYPING:", true);
        }
    };

    useEffect(() => {
        reset(); // Reset idle timer whenever the user types a new message
    }, [currentMessage, reset]);

    return (
        <div>
            <h1>Real-Time Chat</h1>
            <div>
                {messages.map((msg, index) => (
                <div key={index}>{msg}</div>
                ))}
            </div>
            <input
                type="text"
                value={currentMessage}
                onChange={(e) => setCurrentMessage(e.target.value)}
                onInput={handleTyping}
                placeholder="Type a message..."
            />
            <button onClick={handleSendMessage}>Send Message</button>
            {isTyping && <p>Someone is typing...</p>}
        </div>
    );
};

export default ChatApp;

    How It Works:

    • Typing Notification: When a user starts typing in one tab, the TYPING: message is sent to all other tabs, updating their UI with a "Someone is typing..." message.
    • Sending Messages: When a user sends a message, the MESSAGE: message is broadcast to all open tabs, adding the new message to the chat in all tabs.
    • Cross-Tab Sync: All tabs are kept in sync in real-time, and when a user becomes idle, the isTyping status is reset across all tabs.

    In a multi-tab app, you might want to track whether a user is online or offline across all open tabs.

    Scenario:

    • When a user opens a new tab, we want to sync the user's online/offline status across all tabs.
    • If the user becomes idle in one tab, we should update all other tabs to show that the user is offline.
    import React, { useState, useEffect } from "react";
import { useIdleTimer } from "react-idle-timer";

const UserStatusApp = () => {
    const [status, setStatus] = useState("Online"); // User status
    const [messageCount, setMessageCount] = useState(0); // Track messages

    const onIdle = () => {
        setStatus("Offline"); // Set status to offline if the user is idle
    };

    const onActive = () => {
        setStatus("Online"); // Set status to online if the user becomes active
    };

    const onMessage = (msg: string) => {
        // Update status based on received message
        if (msg === "User is offline") {
            setStatus("Offline");
        } else if (msg === "User is online") {
            setStatus("Online");
        }
        setMessageCount((prev) => prev + 1); // Track received messages
    };

    const { message, reset, isLeader } = useIdleTimer({
        onIdle,
        onActive,
        onMessage,
        timeout: 5000, // 5-second idle timeout
        crossTab: true,
        leaderElection: true, // Only the leader tab should send messages
    });

    useEffect(() => {
        if (isLeader()) {
            // Send status update to other tabs when the status changes
            message(status === "Online" ? "User is online" : "User is offline", true);
        }
    }, [status, isLeader, message]);

    return (
        <div>
            <h1>User Status Across Tabs</h1>
            <p>User is currently: {status}</p>
            <p>Messages received: {messageCount}</p>
        </div>
    );
};

export default UserStatusApp;

    How It Works:

    • User Online/Offline Status: The status (Online or Offline) is updated based on idle and active states. The onIdle callback sets the status to "Offline", while the onActive callback sets it to "Online".

    • Cross-Tab Sync: When the status changes, the leader tab broadcasts the status to all other tabs. If a user becomes idle in one tab, all tabs reflect the "Offline" status in real-time.

    • Leader Election: The leaderElection: true ensures that only the leader tab sends the status updates to avoid multiple tabs sending conflicting information.

    In a notification system, you might want to notify all open tabs when a specific event occurs in one tab (e.g., a new notification or user action).

    Scenario:

    • When a new notification is received in one tab, all open tabs should be notified.
    • We also want to ensure that if a notification has been seen in one tab, it is marked as "seen" in all tabs.
    import React, { useState, useEffect } from "react";
import { useIdleTimer } from "react-idle-timer";

const NotificationApp = () => {
    const [notifications, setNotifications] = useState<string[]>([]); // Notifications list
    const [messageCount, setMessageCount] = useState(0); // Track messages

    const onIdle = () => {
        // Handle idle state if necessary
    };

    const onActive = () => {
        // Handle active state if necessary
    };

    const onMessage = (msg: string) => {
        // Handle incoming notifications
        if (msg.startsWith("NEW_NOTIFICATION:")) {
            setNotifications((prev) => [...prev, msg.substring(18)]); // Add new notification
        } else if (msg === "MARK_AS_SEEN") {
            // Mark all notifications as seen
            setNotifications((prev) => prev.map((notif) => `[Seen] ${notif}`));
        }
        setMessageCount((prev) => prev + 1); // Track received messages
    };

    const { message, reset, isLeader } = useIdleTimer({
        onIdle,
        onActive,
        onMessage,
        timeout: 10000, // 10-second idle timeout
        crossTab: true,
        leaderElection: true, // Only one tab should send messages
    });

    const handleNewNotification = () => {
        if (isLeader()) {
        // Send new notification to all tabs
        message("NEW_NOTIFICATION:You have a new message!", true);
        }
    };

    const handleMarkAsSeen = () => {
        if (isLeader()) {
        // Mark notifications as seen in all tabs
        message("MARK_AS_SEEN", true);
        }
    };

    return (
        <div>
            <h1>Multi-Tab Notification System</h1>
            <button onClick={handleNewNotification}>Send New Notification</button>
            <button onClick={handleMarkAsSeen}>Mark All as Seen</button>
            <div>
                <h2>Notifications:</h2>
                {notifications.length === 0 ? (
                <p>No notifications.</p>
                ) : (
                <ul>
                    {notifications.map((notif, index) => (
                    <li key={index}>{notif}</li>
                    ))}
                </ul>
                )}
            </div>
            <p>Messages received: {messageCount}</p>
        </div>
    );
};

export default NotificationApp;

    How It Works:

    • Sending New Notifications: The leader tab sends a "NEW_NOTIFICATION:" message, and all tabs receive this message and update their list of notifications.
    • Marking Notifications as Seen: The leader tab sends a "MARK_AS_SEEN" message, marking all notifications as seen in every tab.
    • Cross-Tab Sync: All tabs stay in sync, ensuring that notifications are consistent across all open tabs.
