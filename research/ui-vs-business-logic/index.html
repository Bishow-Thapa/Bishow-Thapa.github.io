    üß† Rule of Thumb

    • üîÅ If it needs React (like hooks or lifecycle), use a custom hook.
    • üßÆ If it‚Äôs just a computation or decision, use a pure function.

    THEORY

    ‚úÖ Standard Pattern to Separate Business and UI Logic in React

    üß© 1. Component (UI) Layer

    • Purpose: Handles UI rendering and UI-specific behavior (event handlers, form updates, etc.)
    • Contains: useState, useEffect, JSX, component-level logic

    üìÅ Example:

    src/components/ForexForm.js

    üß© 2. Hooks Layer (Optional but Recommended)

    • Purpose: Contains reusable, component-level logic (e.g., form handling, data fetching).
    • Contains: React logic that isn‚Äôt UI, but still uses React (like useEffect or useState)
    • File prefix: use*.js

    üìÅ Example:

    src/hooks/useForexDealLogic.js

    üß© 3. Business Logic Layer

    • Purpose: Pure JS logic ‚Äì no React or UI dependencies. Easy to test.
    • Contains: Decisions, calculations, transformations based on business rules

    üìÅ Example:

    src/business/forexHelpers.js or forexLogic.js

    üß© 4. Services Layer (API / backend interaction)

    • Purpose: Abstract API calls and communication with backend services.
    • Contains: fetch, axios, or any data access logic

    üìÅ Example:

    src/services/forexService.js

    üß© 5. Constants & Types

    • Central place for enums, transaction types, field names, etc.

    üìÅ Example:

    src/constants/forex.js

    Custom Hooks

    Understanding when to use hooks and how they relate to business logic vs UI logic is key to writing scalable React apps.

    ‚úÖ When to Create Custom Hooks

    Custom hooks are useful when you want to encapsulate and reuse stateful logic or side effects. That logic may involve:

    • React state or effects (useState, useEffect, etc.)
    • DOM or event handling
    • Form state handling
    • API calls
    • Lifecycle events tied to UI or state

    üîÑ Hooks Are for React Logic ‚Äî Not Pure Business Logic

    üîç Logic Type Use a Custom Hook? Why?
    UI-related logic (state, effects, etc.) ‚úÖ Yes Hooks encapsulate React-specific logic
    Business rules (calculations, decisions) ‚ùå No These should go into pure functions, not hooks
    Combined logic (e.g. data fetch + UI reset) ‚úÖ Yes Wrap business + UI logic if side effects are needed

    üß© Breakdown: UI Logic vs Business Logic in Hooks

    ‚úÖ Use Hooks For UI Logic When:
    Situation Example
    Side effects need to be reused useFormResetOnTypeChange
    Local state reused across components usePagination, useModal
    React logic is repeated across files useApi, useFetch, useDebounce
    You‚Äôre abstracting interactions with libraries useAntForm, useTableSorting
    ‚ùå Do Not Use Hooks For:
    Situation Example
    Pure functions with no useState or useEffect calculateExchangeRate, getResetFields()
    Stateless rules/validation isUserEligible(deal)

    üß† Rule of Thumb

    • üîÅ If it needs React (like hooks or lifecycle), use a custom hook.
    • üßÆ If it‚Äôs just a computation or decision, use a pure function.

/**************************************************************************************/

// utils/calculateRateChange.ts

export interface RateChange {
  percentage: string;
  difference: string;
}

export function calculateRateChange(
  rateAmount: number,
  currentRate: number
): RateChange {
  const difference = rateAmount - currentRate;
  const percentage = (difference / currentRate) * 100;

  return {
    difference: difference.toFixed(2),
    percentage: percentage.toFixed(2),
  };
}

    ‚úÖ This is a pure function ‚Äî no React state, no side effects ‚Äî perfect for unit tests or reuse.

    Great ‚Äî let's also separate the tolerance calculation into a reusable function and plug it into your existing hook.

    ‚úÖ Step 1: Create a Business Logic Function

    // utils/calculateTolerance.js
export function calculateTolerance(currentRate, tolerancePercent = 5) {
  if (!currentRate || typeof currentRate !== "number")
    return { min: null, max: null };

  const min = currentRate - (currentRate * tolerancePercent) / 100;
  const max = currentRate + (currentRate * tolerancePercent) / 100;

  return {
    min: parseFloat(min.toFixed(4)),
    max: parseFloat(max.toFixed(4)),
  };
}

    You can adjust tolerancePercent as needed (defaults to 5%).

    **‚úÖ Step 2: Update Your Hook to Use It

    // hooks/useRateCalculation.js

  currentRate,
  buyingCurrency,
  sellingCurrency
) => {
  const [rateChange, setRateChange] = useState(null);

    min: null,
    max: null,
  });

    Hooks Testing

    Testing React hooks like useRateCalculation usually involves using React Testing Library‚Äôs @testing-library/react-hooks or the built-in renderHook utility.

    Here‚Äôs a quick step-by-step guide to set it up and test your hook:

    1. Install dependencies

    npm install --save-dev @testing-library/react @testing-library/react-hooks jest

    Note: If you use React 18+, install the compatible version:

    npm install --save-dev @testing-library/react-hooks@next

    2. Create test file for your hook

    features/forex/hooks/__tests__/useRateCalculation.test.js

    import { renderHook } from "@testing-library/react-hooks";
import { useRateCalculation } from "../useRateCalculation";

describe("useRateCalculation hook", () => {
  test("returns null rateChange and toleranceValue when currencies are missing", () => {
    const { result } = renderHook(() =>
      useRateCalculation(100, 100, null, null)
    );

    expect(result.current.rateChange).toBeNull();
    expect(result.current.toleranceValue).toEqual({ min: null, max: null });
  });

  test("calculates rateChange and toleranceValue correctly with valid inputs", () => {
    const { result } = renderHook(() =>
      useRateCalculation(105, 100, "USD", "EUR")
    );

    expect(result.current.rateChange).toEqual({
      difference: "5.00",
      percentage: "5.00",
    });

    expect(result.current.toleranceValue).toEqual({
      min: 95,
      max: 105,
    });
  });

  test("updates values when inputs change", () => {
    const { result, rerender } = renderHook(
      ({ rateAmount, currentRate, buying, selling }) =>
        useRateCalculation(rateAmount, currentRate, buying, selling),
      {
        initialProps: {
          rateAmount: 100,
          currentRate: 100,
          buying: "USD",
          selling: "EUR",
        },
      }
    );

    expect(result.current.rateChange).toEqual({
      difference: "0.00",
      percentage: "0.00",
    });

    rerender({
      rateAmount: 110,
      currentRate: 100,
      buying: "USD",
      selling: "EUR",
    });

    expect(result.current.rateChange).toEqual({
      difference: "10.00",
      percentage: "10.00",
    });
  });
});
